# v0.12.29

## New Features 🎉

* **Compatibility Matrix Update 2026-07-04** ([#3802](https://github.com/pluralsh/console/pull/3802)) by @app/github-actions

  > The compatibility matrix has been updated to reflect the end-of-life date of June 30, 2026, for certain versions of Flux and kube-prometheus-stack. Additionally, kube-prometheus-stack has been upgraded to version 87.6.0, which includes CI pipeline maintenance and non-major dependency updates, ensuring continued stability and support for users. This update is crucial for maintaining compatibility with Kubernetes and ensuring that users are aware of upcoming changes in their deployment environments.

* **Compatibility Matrix Update 2026-07-05** ([#3803](https://github.com/pluralsh/console/pull/3803)) by @app/github-actions

  > The compatibility matrix has been updated to include new versions for both Argo CD and the Kube Prometheus Stack. Notably, Kube Prometheus Stack version 87.10.1 now supports Kubernetes versions 1.25 through 1.35 and includes an updated Alertmanager image tag, enhancing stability and performance. This update ensures users can leverage the latest features and improvements in their Kubernetes deployments.

* **feat: implement vsphere cloud query provider** ([#3805](https://github.com/pluralsh/console/pull/3805)) by @floreks

  > The vSphere cloud query provider has been implemented, allowing users to create and manage connections to vCenter servers directly within the Plural Console. This feature enhances the platform's capabilities by enabling users to query and interact with their vSphere environments, facilitating better management of virtual resources. Additionally, comprehensive setup guides have been added to assist users in configuring their vSphere connections effectively.

* **feat: Fix babysit gaps** ([#3806](https://github.com/pluralsh/console/pull/3806)) by @michaeljguarino

  > The recent update enhances the Plural Console by addressing gaps in the babysitting functionality, ensuring more reliable monitoring and management of agent runs. Key improvements include the addition of a usage summary chip in the agent runs table, allowing users to easily track resource utilization. Additionally, the logs component now features a toggle for displaying metrics charts, providing users with a more flexible and informative logging experience. These changes aim to streamline user interactions and improve overall operational efficiency within the platform.

* **Compatibility Matrix Update 2026-07-07** ([#3807](https://github.com/pluralsh/console/pull/3807)) by @app/github-actions

  > The compatibility matrix for EKS has been updated to include new versions of several addons, such as cert-manager and vpc-cni, along with their compatibility with Kubernetes versions 1.30 through 1.36. This enhancement ensures users can leverage the latest addon features while maintaining compatibility with their existing Kubernetes clusters, improving overall stability and functionality.

* **Compatibility Matrix Update 2026-07-07** ([#3808](https://github.com/pluralsh/console/pull/3808)) by @app/github-actions

  > The compatibility matrix has been updated to include new versions for the Amazon VPC CNI and Traefik, enhancing security and operational capabilities. The Amazon VPC CNI now supports version 1.22.3, which introduces new tunables for `revisionHistory` and `securityContext`, as well as support for Security Groups for Pods on IPv6 clusters. Traefik has been updated to version 3.7.6, which addresses three critical vulnerabilities and introduces operational enhancements to improve reliability and security in Kubernetes environments.

* **feat(github-integration): enhance GitHub toolset with security workflows** ([#3809](https://github.com/pluralsh/console/pull/3809)) by @floreks

  > The GitHub integration has been enhanced to include security workflows, allowing users to access and manage code scanning, Dependabot, and secret scanning alerts directly from the Plural Console. This improvement enables developers to better monitor and respond to security vulnerabilities within their repositories, streamlining the process of maintaining secure codebases. With these new features, users can now leverage the GitHub REST API for a more comprehensive view of their project's security posture.

* **feat: Add timestamp filter in logs and update dependencies** ([#3810](https://github.com/pluralsh/console/pull/3810)) by @maciaszczykm

  > Logs now include a timestamp filter, enhancing the ability to track events over time and improving overall log management. Additionally, several dependencies have been updated to their latest versions, ensuring better performance and security across the application. This update is crucial for users who rely on accurate log data for troubleshooting and monitoring their Kubernetes environments.

* **Compatibility Matrix Update 2026-07-08** ([#3812](https://github.com/pluralsh/console/pull/3812)) by @app/github-actions
