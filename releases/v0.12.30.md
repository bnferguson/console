# v0.12.30

## New Features 🎉

* **Compatibility Matrix Update 2026-07-11** ([#3834](https://github.com/pluralsh/console/pull/3834)) by @app/github-actions

  > The compatibility matrix for EKS has been updated to include new versions for several add-ons, specifically `aws-secrets-store-csi-driver-provider`, `kube-state-metrics`, and `prometheus-node-exporter`. These updates ensure that users can now utilize versions v3.1.1-eksbuild.2, v2.19.1-eksbuild.2, and v1.11.1-eksbuild.7 with Kubernetes versions ranging from 1.30 to 1.36, enhancing compatibility and support for users managing their EKS environments.

* **Compatibility Matrix Update 2026-07-11** ([#3835](https://github.com/pluralsh/console/pull/3835)) by @app/github-actions

  > The compatibility matrix has been updated to include new versions for Karpenter and the Kube Prometheus Stack. Karpenter version 1.13.0 introduces improved handling of IP unavailability and more flexible IAM instance profile creation, enhancing operational efficiency. Additionally, the Kube Prometheus Stack has been upgraded to version 87.14.0, which updates the bundled kube-state-metrics dependency and adjusts PrometheusRule annotations for better alert management. These updates ensure users benefit from the latest features and improvements in their Kubernetes deployments.

* **Compatibility Matrix Update 2026-07-12** ([#3836](https://github.com/pluralsh/console/pull/3836)) by @app/github-actions

  > The compatibility matrix has been updated to include version 87.15.1 of the kube-prometheus-stack, which now supports Kubernetes versions 1.25 through 1.35. This update features an upgraded kube-state-metrics dependency from v7.8.0 to v7.8.1, providing minor fixes and enhancements, ensuring users benefit from improved stability and performance in their monitoring setups.

* **Compatibility Matrix Update 2026-07-13** ([#3837](https://github.com/pluralsh/console/pull/3837)) by @app/github-actions

  > The compatibility matrix has been updated to reflect the new version 1.0.11678 of the wiz-sensor chart, ensuring users have access to the latest features and improvements. This update replaces the previous version 1.0.11492, which may enhance compatibility with Kubernetes versions 1.33 to 1.36. Users can now benefit from the most current functionalities and optimizations provided by this chart version.

* **feat: Followup for agent run workbenches approval flow** ([#3838](https://github.com/pluralsh/console/pull/3838)) by @maciaszczykm

  > The approval flow for agent run workbenches has been enhanced with the addition of a new Hourglass icon, improving the visual representation of pending tasks. This update streamlines the user experience by providing clearer indicators of the status of agent runs, making it easier for users to manage and approve their workflows effectively. These changes are part of ongoing efforts to refine the interface and functionality of the Plural Console, ensuring a more intuitive and efficient user experience.

* **Implement xai provider** ([#3842](https://github.com/pluralsh/console/pull/3842)) by @michaeljguarino

  > The implementation of the xAI provider enhances the AI settings within the Plural Console, allowing users to configure and utilize xAI as an AI provider alongside existing options. This addition includes support for specific configurations such as base URL, model selection, and custom HTTP headers, streamlining the integration of xAI into users' workflows. By expanding the range of AI providers, this update improves flexibility and functionality for users leveraging AI capabilities in their Kubernetes management.

* **Add KubeVirt compatibility scraper** ([#3844](https://github.com/pluralsh/console/pull/3844)) by @KUANPEIEN

  > KubeVirt compatibility has been enhanced with the addition of a new scraper that automatically retrieves and updates compatibility information for various KubeVirt versions against Kubernetes versions. This update includes a dedicated YAML file that outlines supported versions and their compatibility, streamlining the process for users to ensure their KubeVirt deployments align with Kubernetes requirements. This feature is crucial for maintaining operational integrity and simplifying the management of Kubernetes environments.
