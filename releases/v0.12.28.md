# v0.12.28

## New Features 🎉

* **feat: Update agent run UI and approval flow** ([#3736](https://github.com/pluralsh/console/pull/3736)) by @maciaszczykm

  > The agent run user interface has been significantly updated to enhance the approval flow, providing a more intuitive experience for users managing AI agent runs. Key improvements include a redesigned layout for agent run details and the introduction of new icons, which streamline navigation and make it easier to track the status of approvals. These changes aim to improve user efficiency and clarity when interacting with the agent run features.

* **Compatibility Matrix Update 2026-06-28** ([#3780](https://github.com/pluralsh/console/pull/3780)) by @app/github-actions

  > The compatibility matrix has been updated to include version 87.3.0 of the kube-prometheus-stack, which introduces a significant change in how `containers` and `initContainers` fields are rendered using Helm's `tpl` function. This allows for dynamic templating in these fields, enabling more flexible configurations, but users should be cautious as it may lead to unintended template rendering if custom values are not properly managed. Additionally, the update includes routine dependency updates and clarifications on behavior changes to ensure users are aware of potential impacts on their configurations.

* **feat(workbench): redesign job todos and results components** ([#3781](https://github.com/pluralsh/console/pull/3781)) by @floreks

  > The job todos and results components in the workbench have been redesigned for improved usability and visual clarity. The updates include a new timeline layout for todos, making it easier to track progress, and enhanced styling that provides a more spacious and organized presentation of job results. These changes aim to streamline user interactions and enhance the overall experience when managing jobs within the workbench.

* **feat: Tune workbench prompt** ([#3782](https://github.com/pluralsh/console/pull/3782)) by @michaeljguarino

  > The workbench prompt has been refined to require manual approval before creating pull requests, enhancing control over automated changes. This adjustment allows users to provide follow-up prompts to ensure accuracy, thereby improving the overall reliability of the AI-driven automation process. This change is significant for users who rely on precise and validated deployments, as it minimizes the risk of unintended modifications.

* **feat(cloud-query): introduce MetricsLabelSearch functionality** ([#3789](https://github.com/pluralsh/console/pull/3789)) by @floreks

  > The new MetricsLabelSearch functionality has been introduced, enabling users to perform label searches on metrics across various observability tools, including Prometheus, Datadog, CloudWatch, and Azure. This enhancement allows for more granular insights by returning indexed metric tags, which can streamline troubleshooting and improve observability workflows. Users can now efficiently query and filter metrics based on specific labels, enhancing their ability to monitor and manage their Kubernetes environments effectively.

* **feat(workbenches): enhance chart interactivity** ([#3790](https://github.com/pluralsh/console/pull/3790)) by @floreks

  > Chart interactivity in the workbench usage component has been significantly enhanced, allowing users to visualize single data points more effectively. A new crosshair feature and area highlighting for single data points improve the overall user experience, making it easier to interpret data trends and insights. This update is crucial for users relying on precise data analysis within their workbench environments.

* **Compatibility Matrix Update 2026-07-02** ([#3792](https://github.com/pluralsh/console/pull/3792)) by @app/github-actions
